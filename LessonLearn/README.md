# LessonLearn - 기술 보고서 저장소

이 폴더는 PR 해결 후 작성된 기술 보고서를 저장합니다.

## 목적

1. **지식 축적** - 해결한 문제와 배운 점 기록
2. **팀 학습** - 다른 팀원이 유사 문제 해결 시 참고
3. **온보딩** - 새 팀원이 프로젝트 이해에 활용

## 파일 네이밍

```
YYYY-MM-DD_PR{number}_{slug}.md

예시:
- 2026-02-18_PR45_fix-login-error.md
- 2026-02-18_PR46_add-dark-mode.md
```

## 보고서 구조

각 보고서는 다음 섹션을 포함합니다:

### 필수 섹션
- **개요** - 문제 상황과 해결 목표
- **해결 방법** - 코드 변경과 의사결정
- **학습 포인트** - 핵심 배움 3-5개
- **레퍼런스** - 관련 문서/링크

### 학습 포인트 형식

```markdown
### 1. [핵심 개념]
- **무엇**: [개념 설명 1줄]
- **왜 중요**: [중요한 이유 1줄]
- **적용**: [적용 상황 1줄]
```

## 자동 생성

PR이 main 브랜치에 머지되면 GitHub Actions가 자동으로 기술 보고서 초안을 생성합니다.

### 자동 생성 후 할 일
1. [ ] 학습 포인트 상세화
2. [ ] 레퍼런스 링크 추가
3. [ ] 태그 보완

## 검색

보고서를 검색하려면:

```bash
# 키워드로 검색
grep -r "키워드" LessonLearn/

# 태그로 검색
grep -l "#rust" LessonLearn/*.md
```

또는 `/tech-report search {keyword}` 커맨드 사용

## 태그 컨벤션

| 태그 | 설명 |
|------|------|
| #rust | Rust 관련 |
| #frontend | 프론트엔드 관련 |
| #backend | 백엔드 관련 |
| #bugfix | 버그 수정 |
| #feature | 새 기능 |
| #performance | 성능 개선 |
| #security | 보안 관련 |
| #refactor | 리팩토링 |
